Q1
Define Machine Learning. Explain its types with examples.
Definition
Machine Learning (ML) is a branch of Artificial Intelligence (AI) that enables computers to learn from data and improve their performance without being explicitly programmed. It uses algorithms to identify patterns in data and make predictions or decisions automatically.
Types of Machine Learning
1. Supervised Learning

The model is trained using labeled data — both input and the correct output are known. The algorithm maps inputs to outputs by learning from examples.

  • Email Spam Detection — emails labeled Spam / Not Spam
  • House Price Prediction — predict price from size, location, rooms
  • Image Classification — label images as cat, dog, etc.

Algorithms: Linear Regression SVM Decision Tree KNN

2. Unsupervised Learning

The model works with unlabeled data. No correct answer is given — the algorithm finds hidden patterns or groupings in the data on its own.

  • Customer Segmentation — group customers by purchase behavior
  • Market Basket Analysis — find which products are bought together
  • Topic Modeling — automatically group news articles by topic

Algorithms: K-Means DBSCAN PCA

3. Reinforcement Learning

An agent interacts with an environment and receives rewards for correct actions and penalties for wrong ones. It learns the best strategy (policy) by trial and error.

  • Game Playing — Chess AI, AlphaGo, Video Game bots
  • Robot Navigation — robot learns to walk by falling and recovering
  • Self-Driving Cars — learns optimal driving behavior

Q2
What is Feature Engineering? Discuss its importance in the Machine Learning process.
Definition
Feature Engineering is the process of using domain knowledge to select, modify, transform, and create new features (variables) from raw data to improve the performance of a Machine Learning model. It bridges the gap between raw data and a high-performing model.
Key Processes in Feature Engineering
  • 1
    Feature Creation — Creating new features from existing data using domain knowledge. Example: from "Year Built" → "Age of House = Current Year − Year Built"
  • 2
    Feature Transformation — Normalization, Scaling, Log transformation, Encoding categorical data (Label Encoding, One-Hot Encoding).
  • 3
    Feature Extraction — Deriving features from raw data. E.g., extracting edges from images, or using PCA to reduce dimensions.
  • 4
    Feature Selection — Keeping only the most useful features. Removes irrelevant or redundant features to reduce noise.
  • 5
    Feature Scaling — Normalizing values so no feature dominates others. Min-Max (0 to 1), Standard Scaling (mean=0, std=1).
Importance of Feature Engineering
  • Improves model accuracy — better features give the model more useful information
  • Helps model understand patterns — meaningful features highlight underlying relationships
  • Removes irrelevant/noisy data — prevents the model from learning wrong patterns
  • Reduces model complexity — fewer, better features mean simpler, faster models
  • Prevents overfitting — removing redundant features improves generalization to new data
  • Speeds up training — fewer features = less computation time

What is Feature Selection, and why is it important?
Definition
Feature Selection is the process of automatically or manually selecting the most important, relevant features (input variables) from a dataset and removing unnecessary, redundant, or irrelevant features before building a machine learning model.
Methods of Feature Selection
MethodHow It WorksTechniques

Why Feature Selection is Important
  • 1
    Improves Model Accuracy — removing noisy/irrelevant data helps model focus on what matters, improving predictions.
  • 2
    Reduces Training Time — fewer features = less computation. Speeds up both training and inference.
  • 3
    Prevents Overfitting — removing redundant features stops model from memorizing noise in training data.
  • 4
    Makes Model Interpretable — simpler models with fewer features are easier to understand and explain.
  • 5
    Reduces Curse of Dimensionality — high-dimensional data causes models to perform poorly; feature selection reduces this risk.

Q5
What is a Probability Distribution? Provide examples.
Definition
A Probability Distribution is a mathematical function that describes how probabilities are assigned to each possible outcome of a random variable (experiment). It shows how likely each outcome is to occur, and the sum of all probabilities always equals 1.
Probability vs Probability Distribution
ConceptDescriptionExample
ProbabilityLikelihood of a single specific eventP(rolling a 3) = 1/6
Probability DistributionProbability of all possible outcomes of an experimentP(k) = 1/6 for k = 1, 2, 3, 4, 5, 6 (fair die)
Types of Probability Distributions
1. Normal Distribution (Gaussian)
Bell-shaped, symmetric curve. Most values cluster around the mean. Used in natural phenomena.
Example: Heights of people in a population. Most people are near the average height; very tall/short people are rare.
2. Binomial Distribution
Used for experiments with two outcomes (success / failure). Repeated n times with probability p of success.
Example: Probability of getting exactly 3 heads in 5 coin flips.
3. Poisson Distribution
Counts events occurring in a fixed time interval at a constant rate.
Example: Number of customer calls received in 1 hour at a call center.
4. Uniform Distribution
All outcomes are equally likely.
Example: Rolling a fair die — each face has probability 1/6.

Q6
Write the equation for Linear Regression and explain its components.
Definition
Linear Regression is a supervised learning algorithm used to predict a continuous (numerical) output value by establishing a linear relationship between input variable(s) (X) and output variable (Y). It fits a straight line to the data.
Equation — Simple Linear Regression
Simple Linear Regression (1 input variable) y = β₀ + β₁x + ε
y = Dependent variable (value we want to predict)
x = Independent variable (input / predictor feature)
β₀ = Intercept — value of y when x = 0 (where line crosses y-axis)
β₁ = Slope / Coefficient — change in y for every 1-unit increase in x
ε = Error term (difference between predicted and actual value)
Multiple Linear Regression (n input variables) Y = β₀ + β₁x₁ + β₂x₂ + β₃x₃ + ... + βₙxₙ + ε
x₁, x₂, ..., xₙ = Multiple independent variables (features)
β₁, β₂, ..., βₙ = Coefficient for each feature
β₀ = Intercept (bias term)

Q7
What is Cross-Validation? Explain why it is used for model evaluation.
Definition
Cross-Validation is a model evaluation technique that divides the dataset into multiple subsets (folds), trains the model on some folds, and tests on the remaining fold — repeated multiple times. It gives a reliable estimate of how well the model will perform on unseen data.
Types of Cross-Validation
1. K-Fold Cross Validation (Most Common)
Dataset is split into K equal parts. In each iteration, 1 fold is used for testing and (K−1) folds for training. Repeated K times. Final result = average accuracy of all K iterations. Recommended K = 10.
2. Holdout Validation
Dataset split into two parts — typically 80% training, 20% testing (or 50/50). Simple and fast but may miss important patterns in the portion not trained on.
3. LOOCV — Leave-One-Out Cross Validation
Train on n−1 samples, test on 1. Repeated n times (once per data point). Very low bias, high variance. Computationally expensive for large datasets.
4. Stratified K-Fold
Like K-Fold but ensures each fold has the same class distribution as the full dataset. Essential for imbalanced datasets (e.g., 90% class A, 10% class B).

Q8
What is Overfitting? Discuss overfitting and pruning techniques in Decision Trees.
What is Overfitting?
Overfitting occurs when a machine learning model learns the training data too well — including the noise and random fluctuations — and as a result performs poorly on new, unseen data. It has low training error but high testing error. Poor generalization.
Causes of Overfitting
  • Model is too complex — too many parameters relative to data size
  • Too many branches / depth in a Decision Tree
  • Small training dataset — not enough examples to generalize from
  • Training for too many iterations
  • Presence of noisy or irrelevant features
Overfitting in Decision Trees

Decision Trees are especially prone to overfitting. A tree that grows too deep memorizes training examples rather than learning general rules. Every leaf can represent a single data point — perfect on training data but useless on new data.

Pruning — Solution to Overfitting in Decision Trees
Pruning is the technique of removing unnecessary branches and leaf nodes from a fully grown decision tree to reduce overfitting and improve generalization.
Pre-Pruning (Early Stopping)
Stop growing the tree before it becomes too large. Apply stopping criteria during tree construction:
• Set a maximum tree depth (e.g., max_depth = 5)
• Minimum samples required to split a node (e.g., min_samples_split = 10)
• Minimum information gain required to split
Post-Pruning (Bottom-Up Pruning)
Build the full tree first, then remove branches that don't improve performance on validation data:
• Cost Complexity Pruning — penalize tree by number of leaves + error rate
• Reduced Error Pruning — remove nodes that don't increase validation error when removed
• Minimum Description Length Pruning — balance model complexity vs accuracy

Q9
Write a note on Support Vector Machines (SVM).
Definition
Support Vector Machine (SVM) is a supervised machine learning algorithm used primarily for classification tasks (and sometimes regression). It works by finding the optimal decision boundary (hyperplane) that maximizes the margin between two classes in the dataset.
Key Terminology
TermMeaning
HyperplaneThe decision boundary that separates classes. A line in 2D, a plane in 3D, and a hyperplane in higher dimensions.
Support VectorsThe data points closest to the hyperplane. These define the margin and position of the hyperplane.
MarginThe distance between the hyperplane and the nearest support vectors on each side. SVM maximizes this margin.
KernelA function used to map data into a higher dimension so it becomes linearly separable. E.g., RBF, Polynomial, Linear.
Types of SVM
Linear SVM — For data that is linearly separable. A single straight line (hyperplane) can perfectly separate the two classes.
Non-Linear SVM (with Kernel Trick) — For data that cannot be separated by a straight line. A kernel function transforms the data into a higher-dimensional space where it becomes linearly separable. Example: z = x² + y² adds a third dimension.
SVM Diagram
SVM Algorithm Steps
  • 1
    Plot each data point in n-dimensional space (n = number of features)
  • 2
    Find the hyperplane that best separates the two classes
  • 3
    Identify the support vectors (points closest to the hyperplane)
  • 4
    Maximize the margin between the support vectors and the hyperplane
  • 5
    Classify new data points based on which side of the hyperplane they fall on
Applications of SVM
  • Face Detection and Image Classification
  • Text Classification (Spam / Not Spam)
  • Handwriting Recognition
  • Bioinformatics (classifying cancer types from gene data)
  • Intrusion Detection in cybersecurity
Q10
Explain with example: Precision, Recall and F1 Score.
Background — Confusion Matrix

These three metrics are derived from a Confusion Matrix. First, understand its four values:

Actual: PositiveActual: Negative
Predicted: Positive TP — True Positive
Correctly predicted Positive		 FP — False Positive
Wrongly predicted Positive (Type I Error)
Predicted: Negative FN — False Negative
Wrongly predicted Negative (Type II Error)		 TN — True Negative
Correctly predicted Negative

Using example values: TP = 10, TN = 12, FP = 1, FN = 2 (from spam email detector)

1. Precision
Precision measures how many of the predicted positives are actually correct. "Of all emails I predicted as spam, how many were actually spam?"
Precision Formula Precision = TP / (TP + FP)
Example: Precision = 10 / (10 + 1) = 10/11 = 0.909 ≈ 91%
→ 91% of predicted spam emails were actually spam.
2. Recall (Sensitivity)
Recall measures how many of the actual positives were correctly identified. "Of all actual spam emails, how many did I correctly catch?"
Recall Formula Recall = TP / (TP + FN)
Example: Recall = 10 / (10 + 2) = 10/12 = 0.833 ≈ 83%
→ 83% of all actual spam emails were correctly identified.
3. F1 Score
F1 Score is the harmonic mean of Precision and Recall. It balances both metrics and is especially useful when classes are imbalanced. A high F1 score means both Precision and Recall are high.
F1 Score Formula F1 = 2 × (Precision × Recall) / (Precision + Recall)
Example: F1 = 2 × (0.909 × 0.833) / (0.909 + 0.833)
F1 = 2 × 0.757 / 1.742 = 1.514 / 1.742 ≈ 0.87 (87%)
Precision vs Recall Trade-off
Trade-off: Increasing Precision often decreases Recall and vice versa.
• High Precision needed when false positives are costly (e.g., cancer diagnosis — don't flag healthy as sick)
• High Recall needed when false negatives are costly (e.g., fraud detection — don't miss actual fraud)
• F1 Score is used when you need a single balanced metric combining both.

Q11
Differentiate between Linear Regression and Polynomial Regression.
Definitions
Linear Regression: Assumes a straight-line (linear) relationship between the independent variable (X) and dependent variable (Y). Fits a single straight line to the data.

Polynomial Regression: Models a curved (non-linear) relationship by adding higher-degree polynomial terms (x², x³, etc.) of the input feature. It is an extension of linear regression for non-linear data.
Equations
Linear Regression y = β₀ + β₁x
A single predictor term. Produces a straight line.
Polynomial Regression (degree n) y = β₀ + β₁x + β₂x² + β₃x³ + ... + βₙxⁿ
Higher-degree terms curve the line. It is still a linear model in terms of coefficients (β).
Degree 2 = Quadratic, Degree 3 = Cubic, and so on.
